# Plugin Spec — foliage_un06

Foliage / Vegetation painting over Instanced Static Mesh. Replicates Unreal Engine's FoliageType → FoliageTool → HISM pipeline for the Mont-ED engine.

## Identity

- Plugin name: `foliage_un06`
- Slug: `foliage_un06`
- Type: `foliage_tool`
- Protocol preset: `core-bus`
- Agent profile: `generic`
- Version: `0.2.0`

## Intent

Deterministic scatter and interactive brush painting of foliage instances onto terrain surfaces. Uses Poisson-disc sampling for natural-looking distribution with slope, height, and alignment filters. Outputs instance transforms and cluster mesh data for instanced mesh rendering (backs onto `instmesh_un05`).

## Features

- **Poisson-disc sampling** (Bridson's algorithm) — natural non-overlapping distribution
- **Slope filter** — exclude steep terrain (degrees)
- **Height filter** — min/max world height range
- **Align to normal** — instance up-vector follows terrain surface
- **Random rotation** — per-instance yaw variation
- **Random scale** — uniform or per-axis within [min, max]
- **Deterministic** — same seed + same params = same output (xoshiro256**)
- **Cluster output** — spatial cell grouping for ISM batching
- **PluginMeshResult** compatible — scatter/generate_mesh output includes vertex/index counts + bounds

## Data Ownership

- **FoliageType definitions**: owned by plugin (in-memory registry, thread-safe)
- **Instance data**: returned in bus response JSON; caller owns persistence
- **Mesh output**: generated by ClusterBuilder; caller owns vertex/index arrays
- **Temporary files**: none created by default operations
- **Export**: JSON export via standalone GUI saves to `foliage_scatter_export.json`

## Non-Goals

- Do not modify the mont-ed engine source
- Do not implement the `external` protocol (core-bus only)
- Do not implement runtime gameplay foliage interaction
